Deep Transformer Q-Network for Enemy-aware Path Planning with Cover Utilization: Memory‑Augmented Navigation in 2D with Extensions to 3D Point‑Cloud Robotics

Abstract
Problem: Enemy-aware navigation under partial observability from a limited field‑of‑view (FoV)
Approach: A hierarchical two‑level approach: DTQN as a high‑level subgoal selector operating on short observation histories, coupled with a low‑level controller that tracks waypoints with a force-based model
High-level Model I/O, deployment: 2D navigation with visibility/occlusion constraints; extension to 3D via Unity‑ROS and real point‑cloud pipelines
Results: Qualitative improvement over classical planners and RL baselines in success rate, safety margins, and time‑to‑goal

Motivation: Navigation with limited sensing and occlusions; why memory matters
Gap: Classical local planners and memory‑less RL degrade under FoV and occlusion

III. Problem Formulation
Formalize navigation as a POMDP with FoV‑limited observations
State and observation: 2D features for candidate subgoals; occlusion and “enemy visibility” constraints
Action space: discrete subgoal selections (high‑level) and/or low‑level actions (variant)
Reward: goal progress, collision penalty, visibility penalty (being “seen”), action efficiency
Metrics: success rate, time‑to‑goal, path length, min obstacle distance, visibility exposure, collision rate



IV. Method
Hierarchical stack overview:
High level: DTQN‑based subgoal scorer selects a waypoint from the candidate set under FoV and visibility constraints
Low level: trajectory follower (describe this as modular, so any PID, velocity commands can be outputted) that tracks the chosen subgoal and handles short‑horizon collision and enemy avoidance
DTQN architecture and I/O: 
Input is a sequence of k time steps of a 16‑D feature vector per candidate: sequence length k of a 16‑D feature vector [agent_x, agent_y, vel_x, vel_y, goal_dx, goal_dy, dist_goal, cand_x, cand_y, agent_to_cand_dx, agent_to_cand_dy, dist_cand, num_enemies, min_enemy_to_cand_dist, num_enemies_seeing_cand, any_enemy_sees_agent]
Positional encoding for k steps; causal multi‑head attention layers; linear head
Defaults from implementation: d_model=64, n_heads=4, n_layers=3; output shape (B, k, output_dim); we use the last step
Output: scalar Q at the final step used to rank subgoals; Q per candidate; ε‑greedy or argmax (current) selection
Candidate generation in 2D:
Free‑space polygon decomposition and subgoal centroids along safe frontiers; invalidate candidates occluded or too close to obstacles/enemies
Mask rules from implementation: avoid candidates within a small radius of current location (repeat obstacle); avoid candidates farther than the final goal distance
Novelty: visibility‑aware scoring features and exposure‑aware masking integrated into the RL training signal
End‑to‑end system pipeline (2D → 3D):
Perception: 2D map or 3D point cloud→2.5D; persistence filtering of “transient” obstacles
Candidate set: generate safe candidates; compute 16‑D features for each
Control: low‑level controller executes towards chosen subgoal; force↔velocity mapping for ROS cmd_vel; odom feedback closes the loop
Anticipatory safety: predicted enemy FOV unions treated as temporary obstacles to shape low‑level forces
3D extension: point‑cloud to features via clusters projected into the same 16‑D schema for 2D DTQN reuse
Training details (can be brief):
Replay buffer with sequences of length k; target network; MSE loss on TD targets; online updates during rollout with next‑candidate max for bootstrapping.
Reward shaping: goal progress, collision penalty, visibility exposure penalty, control smoothness
Optimization: Adam, learning rate schedule; gradient clipping; mixed‑precision optional; periodic checkpointing and CSV logging of decisions/Qs.
